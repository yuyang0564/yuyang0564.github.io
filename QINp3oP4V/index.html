原型对象/原型链

👉 原型对象

每一个函数都有一个 prototype 属性，它引用了一个对象 , 这个对象就被称为 原型对象 。

原型对象的作用

原型对象中可以包含共享的【属性】与【方法】（由这个函数创建的所有实例）

🗣 "原型对象" 中初始只有一个 constructor 属性， 它默认指向 "当前构造函数"。

🗣 通过该函数生成的实例中会包含一个 【proto】 属性 , 这个属性指向 "构造函数" 的原型对象

🗣 由此得出结论， 【原型对象】 连接了 "构造函数" 和 "实例"

📌 Object.getPrototypeOf( ) 可以返回一个实例指向的 "原型对象"

let num = 1
Object.getPrototypeOf( num )

属性/方法的访问原则

当访问一个实例的 属性/方法 时， 会优先从实例本身开始查找，如果实例中没有，则会去实例的 _ proto _ 所指向的 "原型对象" 中查找。

实例 【属性/方法】的遮蔽性

如果实例存在与 "原型对象" 中同名的"属性/方法" ， 则会遮蔽对"原型对象"中同名"属性/方法"的访问。 只有通过 delete 删除实例中同名"属性/方法"，才能恢复对"原型对象"中同名"属性/方法"的访问

📌 通过实例继承的 hasOwnProperty( ) 可以确定一个属性是在实例还是"原型对象"中 , 它对存在于实例中的属性返回 true

let str = " "
str.hasOwnProperty( constructor )   // false

👉 原型链

由于javascript 是基于原型的继承，任何一个对象都可以充当其他对象的原型，而对象本身又存在一个隐式原型【_ proto _】它也是一个对象指向它的原型对象，这样一直向上追溯就形成了类似链条一样的关系，这就是 "原型链" 。原型链会一直追溯到 Object.prototype , 而它的上层就是Null,也就是"原型链"的尽头

👉 原型链的作用

"原型链" 实现了属性访问的规则，当访问一个 属性/方法 时， 会优先从实例本身开始查找，如果实例中没有，则会去实例的 _ proto _ 所指向的 "原型对象" 中查找 ， 依照这样的规则一直查找到 "原型链"的顶端 。如果未找到，则返回 undefined
