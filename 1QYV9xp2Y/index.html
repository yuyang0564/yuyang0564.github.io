事件

👉 Event 事件对象

浏览器提供了一个 Event 对象，所有的事件都是这个对象的实例 , 它本身是一个构造函数，用来生成新的事件实例

let event = new Event( type , options )
  1. type 参数为事件名
  2. options 是一个可选对象，有 2 个可选配置 （ 如果忽略默认会在“捕获”阶段触发 ）
属性 值
bubbles 事件是否冒泡 true / false (默认)
cancelable 事件是否能被取消 true / false(默认) 
let event = new Event("click",{
    bubbles: true,
    canclelable: false
})

通过 dispatchEvent( ) 用来触发一个事件

let event = new Event("click")
let  div = document.querySelect("div")
div.dispatchEvent(event)  //  div 元素上触发一个 click 事件

👉 事件模型

  1. DOM0 级事件模型
  2. DOM2 级事件模型

区别

DOM0 级事件使用 on 绑定

elementObject.onclick = function() { ... }

🗣 定义的事件在传播时只能 “冒泡”
🗣 在一个对象上绑定多个相同的事件时，最后一个会覆盖之前的
🗣 解除事件时，需要把事件处理函数替换为 “null”

elementObject.onclick = null

DOM2 级事件使用 addEventListener( ) 绑定

elementObject.addEventListener("click", function(){ ... }, false)

🗣 第三个参数默认为 false , 即以 “冒泡”的方式传播 。 设置为 true 时，以 “捕获”的方式传播
🗣 在一个对象上绑定多个相同的事件时，会按照绑定的顺序依次触发
🗣 解除事件时，需要通过 removeEventListener( ) 。

！！ 注： removeEventListener( ) 只能解除 addEventListener( ) 绑定的事件

elementObject.removeEventListener("click", function(){ ... }, false)

👉 事件的传播机制

DOM2 在定义事件时，规定一个事件的传播需经过 3个阶段

  1. 从 window 对象开始逐步向下传播至目标节点 (“捕获阶段”)
  2. 在“目标节点”上触发事件
  3. 从“目标节点”向上传播回 window （“冒泡阶段”）

👉 事件代理

基于事件传播的特性，把原本需要在子元素上响应的事件，委托给上级元素，由上级元素负责监听事件, 通过 event事件对象 中的target属性，判断触发事件的子元素
