Vue 组件

组件是一个带有名称的可复用实例,vue中任何一个应用都是由一个个组件抽象组合而成

它本质上也就是与根实例一样，通过不同的选项，再通过 new Vue() 实例出来的一个新的Vue 对象

静态组件

当组件中的内容都是静态内容时，可以在根元素上设置 v-once 修饰符, 它可以让这些静态内容只计算一次后就被缓存

<div v-once>
  ...
</div>

全局组件

通过Vue实例创建,暴露在全局可用。会增加打包的体积

// 全局组件
Vue.component("child-component",{ ...options })

局部组件

const childComponent1 = { ...options }
const childComponent2 = { ...options }
new Vue({
  components: {
    childComponent1: childComponent1,
    childComponent2: childComponent2
  }
})

单文件组件

使用 .vue 文件方式创建组件,会经过 webpack 编译和打包(生产首选)

组件的 Prop

组件是通过 Prop 来接收外界的数据（详情至 Prop ）

自定义 v-model

v-model 默认会使用名为 value 的 prop , 和 input 事件来进行双向同步。但是在自定义组件中，有可能子组件并没有使用 input 元素,而导致 input 事件无效

  • 父组件
<child-component v-model="user"/>

...
data() {
  return {
    user: "xxxxx"
  }
}
  • 子组件
...
<div @click="handle">{{user}}</div>

...
prop: {
  user: String
}

使用 model 配置选项

通过为子组件设置 model 选项,重新改变绑定的行为

model: {
  prop:   父组件传递的属性名
  event:  用于通知父组件监听的自定义事件
}

以上面为例: 当子组件触发点击事件时，通过向外发送一个click-handle 自定义事件,并抛出新的值

  • 子组件
props: { // 只要接收父组件传递的值 props 是必不可少的
  user: String
},
model: {
  prop: "user",
  event: "click-handle"
},
methods: {
  handle() {
    this.$emit("click-handle","新值")
  }
}
  • 父组件相当于设置如下:
<child-component 
  :user="user"
  @click-handle="user == $event"
/>

自定义 v-model 更好的替代方案 .sync修饰符

  • 父组件
<child-component  :user.sync="user"/>
  • 子组件
...
this.$emit("update:user","新值")

.sync 修饰符相当于

<child-component  
  :user="user"
  @update:user="user = $event"
/>

异步组件

在组件需要被渲染时才会进行加载的组件,通过 import() 函数动态加载

Vue.component("child-component",()=>import("./components/sync-component"))

局部注册

...
components: {
  'child-component': () => import("./components/sync-component")
}
