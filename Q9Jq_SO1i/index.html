跨域

Ajax 通信的主要限制就是跨域的安全策略,它要求发起请求的域名必须与接收请求的域名必须是同源的，属于浏览器保护资源的一种行为

同源策略

同源策略是指: 协议、域名、端口 必须严格一致

"http://www.baidu.com"  "https://www.baidu.com"  // 协议不同导致跨域   

"http://www.baidu.com:8080"  "https://www.baidu.com"  // 端口不同导致跨域

"https://www.baidu.com"  "https://www.baidu.cn"  // 域名不同导致跨域

CROS

浏览器通过“跨域资源共享”解决跨域问题 , 它的基本思路就是使用自定义的HTTP头部允许浏览器与服务器之间相互了解

简单请求: GET 、 POST

当没有自定义请求头,请求类型为text/plain时，浏览器会发送一个额外的请求头 Origin, 它包含了发送请求页面的[协议,域名,端口] , 用来让服务器确定是否为其提供响应

如果服务器为其提供响应,则会再响应头 Access-Control-Allow-Origin 包含相同的源，或者是公共资源时可以为 *

复杂请求

除了上述简单请求之外，其余的都为复杂请求。浏览器会先发送一个 OPTIONS请求也叫做预检请求,请求头中会包含:

  1. Origin 确认源
  2. Access-Control-Request-Method 希望请求的方法
  3. Access-Control-Request-Headers 希望使用的请求头,以逗号分隔

服务器需要返回:

  1. Access-Control-Allow-Origin： 确认源
  2. Access-Control-Allow-Methods：允许的方法（逗号分隔的列表）。
  3. Access-Control-Allow-Headers：服务器允许的头部（逗号分隔的列表）。
  4. Access-Control-Max-Age：缓存预检请求的秒数，再缓存的时间内不会再发送 OPTIONS 请求
